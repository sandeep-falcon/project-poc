Concat Map

This operator maps each source value to an inner observable and concatenates the values from each inner observable sequentially. It waits for each inner observable to complete before moving on to the next one. Use concatMap over mergeMap when order is important for you.

When to Use

You need to process values sequentially (making requests one by one). The order of execution matters, and each value or request should finish before starting the next. You're handling tasks that can't run in parallel (database operations with dependencies).
